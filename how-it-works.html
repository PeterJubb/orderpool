How Order Pool works

Order Pool is a shared manufacturing request platform built on transparency and execution. Publishers publish paid quote requests into a pool. Manufacturers pull work by accepting what fits. Progress updates stay attached to the order and syndicate automatically.

What makes it a pool?

This is the core mechanic

Work is shared

When a publisher submits and pays, their request enters the quote pool. Multiple manufacturers can review it.

Publishers

Create a brief and publish it into the pool as a quote request.

Manufacturers

Browse the pool and accept only what fits their capability and capacity.

Acceptance converts

The moment a manufacturer accepts, the request converts into an order with clear states.

Pool → Order

Acceptance pulls it out of the pool and starts execution.

One source of truth

Order detail becomes the canonical place for progress and updates.

Publisher flow

Draft → submit + pay → in pool → accepted → order
1
Create a brief
Draft what you need: quantity, timeline, constraints, assets. It stays private while drafting.
2
Submit + pay
Publishing converts the brief into a paid quote request. It becomes visible in the pool.
3
In the quote pool
Manufacturers review and decide fit. You wait for acceptance.
4
Manufacturer accepts
Acceptance converts the request into an order. It appears in your Orders tab.
5
Track through fulfillment
You see progress states and a syndicated update stream as the manufacturer executes.
6
Stay aligned
No email chains. Questions and updates stay attached to the order.

Manufacturer flow

Browse pool → accept → manage → update
1
Set capabilities
Define categories, MOQs, lead times, and regions so you pull work that fits.
2
Browse the pool
Review paid requests. Decide quickly based on quantity, constraints, and timeline.
3
Accept a request
Acceptance converts it into an order you own. You commit to execution and updates.
4
Move through states
Accepted → Sampling → Production → Shipping → Fulfilled
5
Post milestone updates
Updates you post syndicate to the publisher dashboard automatically.
6
Finish cleanly
Mark fulfilled when complete. Public tracking can reflect status only.
